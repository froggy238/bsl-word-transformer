# Decision Log

Running record of project decisions, kept alongside the code so every design
choice in the dissertation can be traced to a dated rationale.

| Date | Decision | Reason |
|------------|----------|--------|
| 2026-07-09 | Provisional 50-word vocabulary drafted per the proposal criteria (everyday communicative value, semantic-category coverage, mix of one- and two-handed signs). | Pending an availability check against SignBSL.com clip counts (`python -m src.download --dry-run`) before the list is frozen; words with too few dictionary clips will be substituted. |
| 2026-07-09 | MediaPipe pinned to 0.10.9 using the legacy Solutions API (`mediapipe.solutions.holistic`, `model_complexity=1`). | Holistic is deprecated in the newer Tasks API; pinning the version and API guarantees identical landmark extraction across machines and for the lifetime of the project. |
| 2026-07-09 | Mouth-region subset fixed at 30 face-mesh points: the 20 outer-lip indices plus 10 inner-lip indices. | Captures mouthings and mouth gestures relevant to BSL word discrimination at minimal feature cost, keeping the frame vector at 105 landmarks x 3 = 315 dims. |
| 2026-07-09 | Missing-landmark policy: linearly interpolate gaps of <= 5 consecutive frames per block; longer gaps stay NaN through normalisation and are zeroed only after normalisation (dataset layer). | Short gaps are tracking flicker and safe to in-fill; long gaps are genuine absence (e.g. hand out of frame), so zeros after normalisation act as an explicit "missing" signal without inventing motion. Zeroing before normalisation would corrupt the shoulder-frame statistics. |
| 2026-07-09 | LSTM baseline hidden size 256 (2 layers, unidirectional). | Yields roughly a 12% parameter gap to the ~1M-parameter transformer — within the proposal's "roughly 10%" matched-capacity spirit. Exact trainable-parameter counts are printed at model construction and reported in the dissertation. |
| 2026-07-09 | Vocabulary substitutions after the SignBSL availability sweep: want -> need (2 vs 5 variants), come -> ask (2 vs 11), tomorrow -> morning (2 vs 15). | The methodology requires >= 3 distinct dictionary clips per word; replacements stay in the same semantic category (verbs, verbs, time) as required. Raw sweep output kept in `data/availability_check_2026-07-09.txt`. |
| 2026-07-09 | Contributing organisation attributed from the media URL path (`media.signbsl.com/videos[/bsl]/{org}/...`), yielding 13 distinct organisations across 293 clips. | The sign pages do not mark up per-video publishers reliably; the hosting path is the most consistent per-clip source signal. The organisation column is the signer proxy for the grouped split, so this attribution is load-bearing. |
| 2026-07-09 | Downloaded 6 extra variants for `make` (--max-per-word 12). | Its first six clips all came from one organisation, which would have forced the word to be absent from either train or val under the grouped split. It now spans 4 organisations. |
| 2026-07-09 | Train/val split fixed explicitly: val = corpusngt + deafway + gpnhs (69 clips, 24%), train = the other 10 organisations (224 clips, 76%). One class (`water`) has no val clips. | Exhaustive enumeration of all organisation subsets showed full 50-class val coverage is only achievable at a 69/31 split, which strays too far from the methodology's "approximately 80/20" and cuts training data. The chosen split is the frontier point nearest 80/20 with only one unvalidated class; `water`'s validation behaviour is monitored via train metrics and the held-out test instead. Recorded via `--val-orgs` in `data/splits.json` for exact reproducibility. |
| 2026-07-09 | Hand-landmark dropout accepted as a property of the data: at least one hand is detected in 69.9% of frames overall, varying strongly by source (18% winkball to 100% bslfirst/exeter/wolver); face 100%, pose ~100%. | Three mitigations were tested on the lowest-scoring clips and none moved the needle: 2x frame upscaling, `model_complexity=2`, and detection/tracking confidence lowered to 0.3/0.2 (all within ±1pp of baseline). The dropout is inherent to small, motion-blurred dictionary video under Holistic 0.10.9. The pipeline's gap policy (interpolate <= 5 frames, zero + presence flag beyond) plus always-present pose wrists/elbows carry the movement signal; per-source rates are reported in the EDA as §3.4 requires, and this becomes Discussion material. |
| 2026-07-09 | Hyperparameters frozen at lr 3e-4, weight decay 0.01 after the Chapter 3 selection grid ({3e-4, 1e-3} x {0.01, 0.05}, seed 42, augmented, both architectures, validation only). | 3e-4/0.01 was joint-best for both architectures (transformer 0.130, lstm 0.043); results were nearly flat across the grid. These match the values already committed in the 12 run configs, so the completed 12-run grid stands as the final experiment grid. Grid log kept at `results/lr_grid_log.txt`. |
| 2026-07-09 | 12-run grid completed on laptop CPU (not Colab). Best val acc, mean±std over seeds: Transformer+aug 0.121±0.014, Transformer no-aug 0.145±0.020, LSTM+aug 0.048±0.007, LSTM no-aug 0.048±0.018. Label-shuffle check: 0.029 ≈ chance. | Runs take ~5 min each on CPU, so Colab was unnecessary; device and versions are logged per run in env.txt. Train accuracy reaches ~100% in all runs, so low val accuracy is signer/source transfer, not underfitting — the validation organisations are entirely unseen. This is the honest signer-independent result the proposal's risk register anticipated; provisional reading: RQ1 transformer >> LSTM, RQ2 augmentation does not help transfer at this scale. Final RQ answers come from the one-shot test. |
